Flows, anywhere

Flows are procedures that can be executed by different hosts ‚Äî users (via Obsidian Command Palette), agents (via MCP tooling), and even other flows (via Obsidian's programmatic backend and MIDE flow chaining)!

  • Users: run flows directly from the Command Palette.
  • Agents: invoke flows through MCP tools when reasoning is needed.
  • Other flows: chain across plugins for multi-component prompting.

Distinguish Automation from Reasoning & Save üí≤

Our agents don't have to be super smart (and therefore expensive) if we are. Composable, repeatable, programmatic flows reduce the number of costly model calls and keep outputs consistent. Use agents as reasoning tools ‚Äî not as brute-force automation for everything. Self-hosted means no middleman fees.

  • Only use an agent when reasoning is required. If a step is mechanical, run it as a flow.
  • Engineer better context. Rich vault context makes it harder for agents to go off-track.
  • Chain small, reliable steps. Deterministic sub-flows beat a single giant prompt.
  • Efficient context, always: MIDE-OS automatically avoids duplicate documentation or redundant agent context, so you don't pay more for pointless repetition.

Bottom line: smarter flow design = fewer tokens burned.

Practical wins
  • Batch operations & cache intermediates to avoid repeat calls.
  • Prefer structured prompts + validators to reduce retries.
  • Route tasks: local code ‚Üí flows ‚Üí agent only when needed.
  • Share reusable flows across plugins to amortize cost.
  • Smart context injection tailored for the right host and prompt‚Äînever duplicate what you don't need!

Is MIDE-OS free?

Yes! MIDE-OS core and official plugins are released under BSD Zero Clause license and are free forever. Community-developed plugins may choose their own licenses. We're 100% user-supported‚Äîno VC funding, no ads, no data harvesting.

How do plugins update?

MIDE plugins are Obsidian plugins, so they update seamlessly through GitHub Releases and the Community Plugin interface ‚Äî so updating all your MIDE-powered automations in a single click feels natural.

The SDK and MIDE Core schemas support semantic versioning and are designed to be backwards compatible.

Is it safe?
